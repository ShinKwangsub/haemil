> 상태: 초기 설계 (논의 진행 중)

## 1. 비전

7개 오픈소스 AI 에이전트 플랫폼의 핵심 장점을 **하나의 자연스러운 경험**으로 통합하여,
AI가 비즈니스 파트너 역할을 수행하는 상용 제품을 만든다.

> "아이폰은 터치스크린, MP3, 인터넷, 카메라를 발명한 게 아니라 하나의 경험으로 녹여냈다."
> 우리도 흩어진 기술들을 아름답게 믹스한다.

## 2. 7개 소스 플랫폼 — 각각에서 뭘 가져오는가

### 2.1 claw-code — 코어 엔진의 뼈대
> Claude Code의 Rust 재구현. MIT 라이선스.

**가져올 것:**
| 기술 | 원본 모듈 | 우리 시스템에서의 역할 |
|---|---|---|
| ConversationRuntime 턴 루프 | conversation.rs | 에이전트 실행의 심장 — API 호출→도구 실행→훅→반복 |
| 6단계 명령어 검증 | bash_validation.rs | 모든 시스템 명령어 사전 검증 (Allow/Block/Warn) |
| 5단계 권한 모드 | permissions.rs | 에이전트 권한 계층화 (ReadOnly→Allow) |
| 구조화된 세션 압축 | compact.rs | 장기 세션에서 토큰 효율 + 맥락 유지 |
| 1회 복구 정책 | recovery_recipes.rs | 장애 시 1회 자동 복구 → 실패 시 에스컬레이션 |
| 선언적 정책 엔진 | policy_engine.rs | if/else 하드코딩 대신 선언적 규칙으로 행동 제어 |
| Worker Boot 상태 머신 | worker_boot.rs | 에이전트 라이프사이클 (Spawning→Ready→Running→Done) |
| JSONL 세션 저장 | session.rs | append-only 크래시 복구 |
| 동적 프롬프트 조립 | prompt.rs | 컨텍스트 기반 시스템 프롬프트 빌더 |
| 샌드박스 격리 | sandbox.rs | 실행 환경 격리 |
| MCP 도구 브릿지 | mcp_tool_bridge.rs | 외부 도구 자동 발견/연결 |
| 도구 매니페스트 | tools/lib.rs | 확장 가능한 도구 등록 시스템 |

**적용 방식:** Rust → Go 포팅

---

### 2.2 Hermes — 자기 학습 + 영속 메모리
> Nous Research의 자기 학습형 에이전트. 오픈소스.

**가져올 것:**
| 기술 | 우리 시스템에서의 역할 |
|---|---|
| 자기 학습 루프 | 경험 → 스킬 자동 생성 → 사용 중 개선 → 지식 영속화 |
| 영속 메모리 시스템 | FTS5 전문검색 + LLM 요약으로 세션 간 기억 유지 |
| MCP 프로토콜 지원 | 모든 MCP 서버에 연결하여 도구 무한 확장 |
| Skills Hub 표준 | agentskills.io 호환 스킬 공유/설치 |

**왜 독보적인가:** 23개 플랫폼 중 자기 학습 루프는 Hermes만 가지고 있음. 최대 차별화 요소.

**적용 방식:** 철학 차용 + Go로 재구현

---

### 2.3 OpenClaw — 멀티채널 게이트웨이
> AI 에이전트 게이트웨이. 오픈소스. 68K+ stars.

**가져올 것:**
| 기술 | 우리 시스템에서의 역할 |
|---|---|
| 24+ 메시징 채널 통합 | WhatsApp, Telegram, Slack, Discord, iMessage, Teams, LINE 등 어디서든 접근 |
| 자율 스킬 생성 | 필요한 스킬을 에이전트가 스스로 코드 작성하여 생성 |
| 프로액티브 자동화 | 사용자 선호도 장기 기억 → 선제적 작업 수행 |

**왜 독보적인가:** 대부분 CLI/웹 1-2개 인터페이스만 지원. 24+채널은 유일.

**적용 방식:** 철학 차용 + Go로 재구현

---

### 2.4 Paperclip — 조직 관리 + 거버넌스
> AI 에이전트 오케스트레이션 플랫폼. 오픈소스. 38K+ stars.

**가져올 것:**
| 기술 | 우리 시스템에서의 역할 |
|---|---|
| 조직 구조 모델링 | CEO → 매니저 → IC 체계로 에이전트 팀 구성 |
| 거버넌스 시스템 | 승인 게이트, 예산 관리, 감사 추적, 롤백 |
| 하트비트 패턴 | 에이전트가 5~60분 간격으로 깨어나 작업 확인 후 실행 |
| Atomic Checkout | 태스크 체크아웃과 예산 집행이 원자적 → 중복 작업 방지 |
| 영속적 상태 | 하트비트 간 동일 태스크 컨텍스트 유지 |

**왜 독보적인가:** 예산/승인/감사까지 하는 에이전트 오케스트레이션은 유일.

**적용 방식:** 철학 차용 + Go로 재구현

---

### 2.5 GoClaw — 고성능 + 보안
> OpenClaw를 Go로 재작성. 오픈소스.

**가져올 것:**
| 기술 | 우리 시스템에서의 역할 |
|---|---|
| Go 네이티브 아키텍처 | 단일 바이너리 ~25MB, 1초 미만 시작, goroutine 동시성 |
| 5계층 보안 | Rate limiting, 인젝션 탐지, SSRF 방어, 셸 패턴 매칭, AES-256-GCM |
| RLS 멀티테넌트 | PostgreSQL Row-Level Security로 조직 간 데이터 완전 격리 |
| 20+ LLM 프로바이더 | OpenAI, Anthropic, Google, Groq, Mistral, Ollama 등 |
| 프롬프트 캐싱 | 최대 90% 비용 절감 |

**왜 핵심인가:** Python 편중 생태계에서 유일한 Go 기반. 우리 코어 언어와 동일.

**적용 방식:** 구조 차용 + 직접 참고하며 Go 구현

---

### 2.6 Goose — MCP 네이티브 + 로컬 퍼스트
> Block(Square)이 만든 AI 코딩 에이전트. Apache 2.0. Rust 코어.

**가져올 것:**
| 기술 | 우리 시스템에서의 역할 |
|---|---|
| MCP 네이티브 지원 | Model Context Protocol로 외부 도구를 플러그인처럼 연결 |
| 로컬 퍼스트 설계 | 로컬 모델(Ollama 등)과 클라우드 모델을 자유롭게 교체 |
| 확장 프로토콜 | 에이전트가 DB, API, 파일시스템 등에 표준화된 방식으로 접근 |

**왜 가져오나:** claw-code의 MCP 브릿지와 결합하면 도구 통합 레이어가 완성됨.

**적용 방식:** 패턴 차용 (Apache 2.0 ✅)

---

### 2.7 AutoAgent — 에이전트가 에이전트를 만든다
> AI가 에이전트를 스스로 설계/최적화하는 메타 에이전트. MIT.

**가져올 것:**
| 기술 | 우리 시스템에서의 역할 |
|---|---|
| 메타 에이전트 자동 최적화 | 프롬프트, 도구 구성, 오케스트레이션을 자동으로 개선 (hill-climbing) |
| program.md 단일 파일 에이전트 정의 | 마크다운 하나로 "어떤 에이전트를 원하는지" 기술 → AI가 구현 |
| Docker 격리 실행 | 에이전트 자율 수정 시 호스트 보호 |

**왜 가져오나:** Hermes의 "자기 학습"이 스킬 레벨이라면, AutoAgent는 "에이전트 자체"를 최적화.

**적용 방식:** 철학 차용 (MIT ✅)

## 3. 믹스 설계 — 7개가 하나로 녹아드는 방식

### 3.1 레이어별 책임 매핑

```
+---------------------------------------------------+
|  사용자: "매출 분석 보고서 만들어줘"                   |
+---------------------------------------------------+
           │
           ▼
+---------------------------------------------------+
|  Gateway (OpenClaw + GoClaw)                       |
|  어떤 채널에서든 받아서 라우팅                         |
|  - 디스코드/텔레그램/슬랙/웹 → 통합 메시지 포맷        |
|  - 20+ LLM 프로바이더 중 최적 모델 선택               |
|  - 프롬프트 캐싱으로 비용 절감                        |
+---------------------------------------------------+
           │
           ▼
+---------------------------------------------------+
|  Orchestration (Paperclip)                         |
|  누가 이 일을 할지 결정                               |
|  - 조직 구조에서 적합한 에이전트/팀 배정               |
|  - 예산 확인 + Atomic Checkout                      |
|  - 승인 필요 시 → 사용자에게 알림                     |
+---------------------------------------------------+
           │
           ▼
+---------------------------------------------------+
|  Agent Runtime (claw-code + Hermes + AutoAgent)    |
|  실제로 일을 한다                                    |
|  - claw-code 턴 루프로 실행                          |
|  - 정책 엔진으로 행동 제어                            |
|  - Hermes식 메모리로 "지난번 선호" 반영               |
|  - AutoAgent로 에이전트 성능 자동 최적화               |
|  - max_iterations로 무한 루프 방지                    |
|  - 장애 시 1회 복구 → 실패 시 에스컬레이션             |
+---------------------------------------------------+
           │
           ▼
+---------------------------------------------------+
|  Security (claw-code + GoClaw)                     |
|  모든 행동을 검증한다                                 |
|  - 6단계 명령어 검증 (실행 전)                        |
|  - 5단계 권한 모드 확인                               |
|  - 5계층 보안 (인젝션/SSRF/Rate limit)               |
|  - 샌드박스 격리                                     |
+---------------------------------------------------+
           │
           ▼
+---------------------------------------------------+
|  Tools (claw-code + Goose)                         |
|  외부 세계와 상호작용                                 |
|  - MCP로 외부 도구 연결 (DB, API, 파일 등)            |
|  - 도구 매니페스트로 어떤 도구든 등록                   |
|  - 로컬/클라우드 모델 자유 교체                        |
+---------------------------------------------------+
           │
           ▼
+---------------------------------------------------+
|  Data (claw-code + GoClaw)                         |
|  모든 것을 기록한다                                   |
|  - JSONL 세션 저장 (크래시 복구)                      |
|  - PostgreSQL + RLS (멀티테넌트 격리)                 |
|  - 세션 압축 (장기 작업 안정성)                        |
+---------------------------------------------------+
           │
           ▼
+---------------------------------------------------+
|  사용자: 결과를 디스코드/텔레그램/웹에서 받는다          |
+---------------------------------------------------+
```

### 3.2 레이어 간 인터페이스 (접점 설계)

| 접점 | 상위 → 하위 | 핵심 계약 |
|---|---|---|
| Gateway → Orchestration | `Message{channel, user, content}` → `Task{assignee, budget, priority}` | 메시지를 태스크로 변환 |
| Orchestration → Runtime | `Task{agent, context, tools}` → `TurnResult{output, usage, status}` | 태스크를 실행하고 결과 반환 |
| Runtime → Security | `ToolCall{name, input}` → `ValidationResult{allow/block/warn}` | 매 도구 실행 전 검증 |
| Runtime → Tools | `ToolRequest{name, params}` → `ToolResponse{result, error?}` | MCP 표준으로 도구 호출 |
| Runtime → Data | `Event{type, payload, timestamp}` → append to JSONL | 모든 행동을 이벤트로 기록 |
| Runtime ↔ Memory | `Query{context}` ↔ `Memory{relevant_history}` | 관련 기억 검색/저장 |

### 3.3 사용자 경험 시나리오

**시나리오 1: "매출 분석 보고서 만들어줘"**
```
1. [Gateway]  디스코드에서 메시지 수신
2. [Orch]     "데이터 분석가" 에이전트 배정, 예산 체크
3. [Runtime]  턴 루프 시작
4. [Memory]   "지난번엔 막대 그래프 선호" 기억 반영
5. [Security] Google Sheets 접근 권한 확인 → Allow
6. [Tools]    MCP로 Google Sheets 데이터 가져오기
7. [Runtime]  데이터 분석 + 차트 생성 + 보고서 작성
8. [Security] 파일 쓰기 검증 → Allow
9. [Data]     전 과정 JSONL로 기록
10. [Gateway] 디스코드로 보고서 전송
```

**시나리오 2: 사용자 없이 밤새 작업**
```
1. [Orch]     하트비트 → 백로그에서 "API 리팩토링" 태스크 발견
2. [Orch]     "백엔드 개발자" 에이전트 배정, Atomic Checkout
3. [Runtime]  claw-code 턴 루프로 코드 수정
4. [Security] git push 전 권한 확인 → 승인 필요 → 대기
5. [AutoAgent] 실행 중 에이전트 성능 모니터링 + 프롬프트 자동 튜닝
6. [Memory]   작업 내용 + 결정 사항 영속 저장
7. [Data]     세션 압축 (100K 토큰 초과 시)
8. [Gateway]  아침에 텔레그램으로 "3개 태스크 완료, 1개 승인 대기" 알림
```

## 4. 확정 사항

### 4.1 기술 스택
| 항목 | 선택 | 근거 |
|---|---|---|
| 코어 엔진 | **Go** | 성능+구현속도 균형, 단일 바이너리, GoClaw 검증 |

| DB | **PostgreSQL** (내장) | RLS 멀티테넌트 |
| 세션 저장 | **JSONL** (append-only) | claw-code 패턴 |
